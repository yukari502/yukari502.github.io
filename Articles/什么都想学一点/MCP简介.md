# Model Context Protocol (MCP) 简介

## 定义

Model Context Protocol (MCP) 是一种协议，允许软件系统通过统一的接口与外部服务器进行通信，从而扩展自身的功能。这些外部服务器被称为 MCP 服务器，它们可以提供工具（Tools）和资源（Resources）。

MCP 服务器在非交互式环境中运行，这意味着它们不能启动 OAuth 流程、打开浏览器窗口或在运行时提示用户输入。所有凭据和身份验证令牌必须通过 MCP 设置配置中的环境变量预先提供。

## 用途

MCP 的主要用途是赋予 AI 助手访问外部系统、执行计算以及在现实世界中采取行动的能力。通过集成 MCP 服务器，AI 助手可以：

1.  **扩展功能**：MCP 允许 AI 助手通过接入各种外部服务（例如天气 API、数据库、GitHub 等）来执行超出其核心能力的任务。
2.  **获取实时数据**：MCP 服务器可以作为数据源，提供实时信息或访问特定数据集，这些数据可以作为上下文供 AI 助手使用。例如，一个天气 MCP 服务器可以提供特定城市的天气预报。
3.  **执行操作**：工具（Tools）是 MCP 的核心功能之一，它使 AI 助手能够调用外部程序或服务来执行特定的操作。这些操作可以是复杂的计算、系统配置更改、数据处理，甚至是通过 API 与外部服务（如 GitHub API 创建 Issue）进行交互。
4.  **模块化和可插拔性**：MCP 提供了一种标准化的方式来添加和管理新的功能模块。开发者可以根据需要创建和部署不同的 MCP 服务器，从而使 AI 助手的架构更加灵活和可扩展。
5.  **增强用户体验**：通过 MCP，AI 助手可以更智能、更高效地响应用户请求，例如，直接提供实时天气信息、管理项目任务或与第三方应用集成。

## MCP 服务器的组成部分

MCP 服务器可以暴露以下两种核心能力：

*   **资源 (Resources)**：代表任何 UTF-8 编码的数据，例如数据库记录、API 响应、日志文件等。资源可以是静态的（通过固定 URI 访问）或动态的（通过 URI 模板访问）。
*   **工具 (Tools)**：表示可执行的功能，允许服务器向系统暴露可操作性。工具能够与外部系统交互、执行计算并采取实际行动。与资源不同，工具通常涉及动态操作，并且能够修改状态。在实际应用中，通常优先使用工具，因为它们提供更大的灵活性，能够处理动态参数。

## 部署与管理

部署 MCP 服务器通常涉及以下步骤：

1.  **项目初始化**：使用如 `@modelcontextprotocol/create-server` 工具来引导一个新的 MCP 服务器项目。
2.  **代码实现**：根据所需功能实现服务器逻辑，定义工具和资源。
3.  **编译构建**：将源代码编译成可执行文件。
4.  **配置集成**：将 MCP 服务器的配置信息（包括执行命令和必要的环境变量，如 API 密钥）添加到客户端的 MCP 设置文件（例如 `cline_mcp_settings.json`）。
5.  **激活与使用**：配置完成后，系统会自动运行 MCP 服务器，并暴露其提供的工具和资源，供 AI 助手调用。

通过 MCP，AI 助手能够打破其自身能力的边界，通过与外部世界的无缝集成，提供更加强大和多样化的服务。
